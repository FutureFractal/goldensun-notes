```sh
# Compiler-generated .s intermediates: Make's %.o:%.c rule writes a .s next to
# every src/ .c, so refresh_expected.sh / a full `make` litters src/ with them.
# Ignore all .s under src/ EXCEPT the hand-written glue (crt0/exports/imports
# and the lib asm call_via/m4a0), which have no .c sibling and are tracked
# source-of-truth.
src/**/*.s
!src/**/crt0.s
!src/**/exports.s
!src/**/imports.s
!src/**/call_via.s
!src/**/m4a0.s
!src/**/agb_flash_data.s

# The cross-dir rule (asm/%.o: src/%.c) regenerates asm/<bank>/X.s for every
# matched src/<bank>/X.c — build artifacts, not source-of-truth. Ignoring them
# does NOT untrack the committed raw disassembly .s (those stay tracked since
# git never untracks tracked files). NOTE: a genuinely new raw-disassembly .s
# (e.g. a fresh split) must be added with `git add -f`.
asm/**/*.s
```

You can find `GameStart` in https://github.com/Coaltergeist/goldensun-decomp/blob/5bd24497cfb52326c30ca996da3a38d067b3b58d/asm/rom_8a000/rom_8a5f8_a_c_a.s. It is not a build artifact.

Yet in `.gitignore` we have `asm/**/*.s`. If we modify it, git recognizes it as tracking probably manually added so it is an exception to the rule. Even if we add `asm/**/*.s` as a rule we do need to remove some files that aren't built which are an exception.

The hint in the above comment is that there are "genuine raw-disassembly .s"s that one may explicitly opt-in to be tracked with `git add -f`.

So when we we encounter an `extern` function declaration, look for the corresponding code under `asm/*`.
